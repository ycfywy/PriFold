## 1. v10 现状诊断

### v10 并没有失败——只是训练不够

| 对比点 | v9 @ epoch 80 | v10 @ epoch 80 | 差异 |
|--------|--------------|----------------|------|
| val F1 | 0.6447 | **0.6629** | v10 +0.018 ✅ |
| val F1 @ best | 0.6814 (e160) | 0.6629 (e80) | v10 还没到 e160 |
| loss | 3.408 | **3.216** | v10 收敛更快 ✅ |
| test F1 | — | 0.6698 | — |

**结论：v10 在相同 epoch 下比 v9 更好（+0.018），但因为 OOM 频繁中断、max_sq_tokens 被迫从 600K 降到 200K，每 epoch 训练效率只有 v9 的 1/3。v10 需要继续训练到 200 epoch 才能公平对比。**

### v10 的核心问题：显存

| 配置 | v9 | v10 |
|------|-----|------|
| max_sq_tokens | 600,000 | 200,000 |
| batch_size | 12 | 4 |
| grad_accum | 2 | 6 |
| 每 epoch 有效样本 | ~同 | ~同 |
| 每 epoch 时间 | ~237s | ~275s |
| MARS 梯度 | ❌ 无 | ✅ 后 2 层 27M |

MARS unfreeze 导致每个 batch 需要额外 ~20GB 显存存储梯度和激活值。

## 3. 为什么 MARS Unfreeze 没有如预期大幅提升

### 原因分析

1. **训练不够**：v10 到 epoch 80 的 val F1=0.6629，v9 在 epoch 80 是 0.6447。v10 收敛曲线更好，只是还没训够
2. **显存瓶颈**：max_sq_tokens 被迫从 600K 降至 200K，batch 多样性降低
3. **MARS LR 过于保守**：1e-5 可能太低，MARS 后 2 层变化很慢
4. **长序列梯度问题**：MARS 在长序列上 attention map 很大，梯度计算消耗巨大

---

## 4. 改进方向（按优先级）

### 方案 A：让 v10 跑完 200 epoch（最简单）

- v10 在同 epoch 下已经比 v9 好
- 如果 v10 延续趋势，epoch 160 的 val F1 预计 0.69-0.70，test F1 可能 0.71+
- **行动：让训练继续跑到 200 epoch**
- 风险：低，但需要时间

### 方案 B：提高 MARS LR（中等收益，低成本）

- 当前 mars_lr=1e-5 太保守
- 建议提高到 **3e-5** 或 **5e-5**，配合 warmup=15 保护
- 预期：加速 MARS 层适配，可能提前 20-30 epoch 收敛
- **行动：修改 config 中 mars_lr，重新开训**

### 方案 C：只 Unfreeze 最后 1 层（节省显存）

- 2 层 unfreeze = 27M 可训参数，显存大
- 1 层 unfreeze = ~13M，显存压力减半
- 可以恢复 max_sq_tokens=400K，batch 多样性更好
- **行动：修改 unfreeze_last_n=1**

### 方案 D：MARS 层使用 Gradient Checkpointing（高级）

- 只存 MARS 层的输入/输出，不存中间激活
- 时间换空间，可以在 200K tokens 下训练 2 层 unfreeze 而不 OOM
- 需要改 mars_forward_with_attn 代码

### 推荐组合：**B + C（提高 MARS LR + 只解冻 1 层）**

这样可以：
- 恢复 max_sq_tokens=400K（更好的 batch 多样性）
- 更大的 MARS LR 加速适配
- 参数量减半，训练更稳定

---

## 5. 具体行动计划

### v10 继续训练（当前已在跑）

让它跑到 200 epoch 看最终结果，作为 baseline。

### v10b 新实验：Unfreeze 1 层 + 提高 LR

```json
{
  "v10": {
    "freeze_mars": "partial",
    "unfreeze_last_n": 1,        // 2→1
    "mars_lr": 3e-5              // 1e-5→3e-5
  },
  "training": {
    "max_sq_tokens": 400000,     // 200K→400K (恢复)
    "batch_size": 8,             // 4→8
    "gradient_accumulation_steps": 3,  // 6→3
    "warmup_epochs": 15
  }
}
```

预期效果：
- 每 epoch 速度接近 v9（~240s vs v10 的 275s）
- MARS 适配更快（LR 3x）
- 目标：test F1 > 0.71

---

## 6. 关键结论

1. **MARS Unfreeze 方向是对的** — v10 同 epoch 对比 v9 更好
2. **当前 v10 的问题是训练效率**，不是方法本身
3. **解冻 1 层 + 提高 LR + 恢复大 batch** 是最优平衡点
4. **v10 继续跑着，同时开 v10b 新实验**
