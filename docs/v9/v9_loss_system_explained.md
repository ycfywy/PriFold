# PriFold v9 Loss 系统详解

## 为什么需要这么多 Loss？

RNA 二级结构预测本质是一个**极度不平衡的二分类问题**：

- 一条长 200 的 RNA，有效候选位置 = 200×200/2 ≈ 20000 个
- 真实配对通常只有 ~50 对（密度 ~0.25）
- **正负比例约 1:400**

单一 BCE loss 在这种场景下完全失效——模型只要全预测"不配对"就能获得 99.75% 准确率。所以我们需要多个 loss 从不同角度引导模型。

---

## Loss 全景图

```
Total Loss = BCE主分类 + 区域重叠 + 数量约束 + 精度惩罚 + 偏移奖励
              ↓            ↓           ↓           ↓           ↓
           [1] BCE      [2] Dice    [4] Count   [6] FP      [8] Shift
           [3] DST                  [5] Density [7] BP
```

分为 **4 个功能组**：

| 功能组 | 解决什么问题 | 包含哪些 Loss |
|--------|-------------|--------------|
| **主分类** | 让模型学会区分配对/不配对 | BCE + Dice + DST |
| **数量控制** | 控制预测总数别太多或太少 | Pair Count + Density Head |
| **精度提升** | 减少错误预测(FP) | FP Penalty + BP Compat |
| **容错奖励** | 对"差一点对"的预测宽容些 | Shift Loss |

---

## 逐个解释

### [1] Focal BCE + OHEM — 主力分类 Loss

**一句话**：告诉模型每个位置是不是配对，对难分的负样本加重惩罚。

```
作用：基本的"对/错"判断
权重：1.0（主 loss）
```

**问题**：20000 个候选中只有 50 个是正，如果所有负样本都算 loss，梯度被海量简单负样本淹没。

**解决**：
- **Focal**：对"已经学会"的简单样本降权（γ=1.0）
- **OHEM**：只保留最难的 top-k 个负样本（k = 正样本数 × 3）
- **Pos weight**：正样本加权（自适应，约 50-99 倍）

### [2] Dice Loss — 区域重叠度

**一句话**：从集合角度看预测和 GT 的重叠程度。

```
作用：看"整体有多少配对被找到了"
权重：0.5
公式：1 - 2|P∩G| / (|P|+|G|)
```

**为什么需要**：BCE 是逐像素的，Dice 是看全局重叠，对类不平衡更鲁棒。两者互补。

### [3] DST (Density-Stratified Tversky) — 低密度样本保护

**一句话**：对配对很少的 RNA（density<10%），额外惩罚漏检。

```
作用：保护稀疏样本不被忽略
权重：0.4
```

**逻辑**：
- 配对密度 < 10% 的样本：α=0.7, β=0.3 → 更惩罚 FP（精确率优先）
- 配对密度 ≥ 10% 的样本：α=β=0.5 → 平衡

### [4] Pair Count + Ratio Penalty — 数量约束

**一句话**：约束预测的总配对数接近 GT 数量。

```
作用：防止过预测/欠预测
权重：Count=0.3, Ratio=0.2
```

- **Pair Count**：`smooth_L1(pred_density, gt_density)` — 预测密度和真实密度要接近
- **Ratio Penalty**：`ReLU(pred/gt - 1.20)` — 如果预测比 GT 多 20% 以上就惩罚

### [5] Density Head — 辅助密度预测

**一句话**：教模型先学会"这条 RNA 大概有多少配对"。

```
作用：为推理时的 budget 提供参考
权重：0.3
公式：MSE(predicted_density, gt_density)
```

这不直接改善分类，而是为推理时的 top-k 选择提供 budget 估计。

### [6] FP Penalty — 精确率提升

**一句话**：对模型预测了"配对"但实际不是的位置，额外加罚。

```
作用：降低 False Positive
权重：2.0
```

**为什么 BCE 不够**：BCE 的负样本 loss 被 OHEM 稀释了，FP Penalty 专门针对"模型已经预测为正但实际为负"的位置单独加罚。

### [7] BP Compatibility — 碱基配对合法性

**一句话**：RNA 只有 A-U、G-C、G-U 三种合法配对，预测非法配对要惩罚。

```
作用：利用生物先验过滤非法预测
权重：0.3
```

比如模型预测了 A-A 配对，这在生物学上不可能，直接惩罚。

### [8] Shift-Aware Loss — 偏移容错

**一句话**：如果预测偏了 1-2 格但方向对了，给奖励而不是惩罚。

```
作用：对"差一点正确"的预测宽容
权重：-0.6（负号=奖励，降低 total loss）
radius：2（±2 格内算 near-miss）
```

**背景**：v8 分析发现 33% 的 FP 是 shift±1（差一格就对了）。这些不应该和"完全乱猜"的 FP 同等惩罚。

## 各 Loss 的贡献占比（典型训练中期）

```
BCE (OHEM)     ████████████████████  ~40%   ← 主力
Dice           ████████             ~15%
DST            ██████               ~12%
FP Penalty     ██████               ~12%
Pair Count     ████                 ~8%
Density Head   ███                  ~6%
BP Compat      ██                   ~4%
Shift (奖励)   ██ (负)              ~-3%   ← 降低总 loss
```

---

## 能不能减少？

可以。按重要性排序：

| 等级 | Loss | 去掉会怎样 |
|------|------|-----------|
| **必须** | BCE + OHEM | 没有主分类 loss 模型无法学习 |
| **必须** | Dice | 去掉后 F1 下降 ~3% |
| **重要** | FP Penalty | 去掉后 Precision 下降 ~5% |
| **重要** | Shift Loss | 去掉后偏移预测增加，F1 下降 ~2% |
| **有用** | DST | 去掉后低密度样本表现变差 |
| **有用** | Pair Count | 去掉后 pred/gt ratio 失控 |
| **辅助** | Density Head | 去掉后推理 budget 估计不准 |
| **辅助** | BP Compat | 去掉影响小（生物先验） |

**最小可行组合**（4 个）：BCE+OHEM + Dice + FP Penalty + Shift

**完整组合**（8 个）：精度最高，各 loss 解决不同问题，互不冲突。

## 类比理解

把训练想象成教一个人找 RNA 配对：

1. **BCE**：「这个位置对不对？」— 基本判断
2. **Dice**：「整体找到了百分之几？」— 全局评估
3. **DST**：「配对很少的 RNA 也别放弃！」— 照顾弱势
4. **Pair Count**：「别贪心找太多，也别偷懒找太少」— 数量管控
5. **Density Head**：「先估计大概有多少对，心里有数」— 全局感知
6. **FP Penalty**：「说错了要加倍扣分！」— 精确率压力
7. **BP Compat**：「A 和 A 不能配对，这是常识」— 生物规则
8. **Shift Loss**：「差一格算你半对，别气馁」— 容错鼓励

## 推理阶段参数

训练完成后，推理时不再计算 loss，而是用一套**筛选策略**从模型输出的 score map 中挑出最终预测：

### 推理流程

```
模型输出 score map (L×L, 每个位置 0~1 的配对概率)
    ↓
[1] Score Threshold 过滤 → 低于 0.43 的直接丢弃
    ↓
[2] BP Compatibility 过滤 → 非 A-U/G-C/G-U 的位置丢弃
    ↓
[3] Density Budget 估计 → 模型预测"这条RNA大概有N对"
    ↓
[4] Length Decay 调整 → 长序列收紧 budget（但有 floor 兜底）
    ↓
[5] Top-K 选择 → 从剩余候选中取分数最高的 K 对
    ↓
[6] 对称化 → (i,j) 配对则 (j,i) 也标记
    ↓
最终预测 contact map
```

### 参数详解

| 参数 | v9 值 | 含义 |
|------|-------|------|
| `score_threshold` | **0.43** | 模型输出 < 0.43 的位置直接过滤，不参与后续选择 |
| `use_density_budget` | true | 用 density head 预测的密度来估计最大配对数 |
| `default_budget_fraction` | 0.30 | 备用：如果不用 density budget，取 L×30% 作为最大配对数 |
| `length_decay` | **0.15** | 长序列收紧系数：`factor = (100/L)^0.15` |
| `budget_floor` | **0.6** | factor 最低不低于 0.6，防止长序列被压太狠 |
| `bp_compat_in_inference` | true | 推理时过滤非标准碱基配对 (A-A, C-C 等) |

### 关键公式：最大配对数 K

```
K = density_pred × L_eff × length_factor × 1.05

其中：
  density_pred = density head 输出（0~1，表示配对密度）
  L_eff = 有效序列长度
  length_factor = max((100/L)^0.15, 0.6)   ← 带 floor 的长度衰减
  1.05 = 5% 余量
```

### 长度衰减效果对比

| 序列长度 | v8 factor (decay=0.3) | v9 factor (decay=0.15, floor=0.6) |
|----------|----------------------|-----------------------------------|
| 50 | 1.23 | 1.11 |
| 100 | 1.00 | 1.00 |
| 200 | 0.81 | 0.90 |
| 300 | 0.72 | 0.84 |
| 400 | 0.66 | 0.79 |
| 490 | 0.62 | 0.75 |

v8 在 490 长度时 budget 被压缩到 62%，v9 保持在 75%，长序列 recall 不再崩塌。

### 为什么不直接取所有 > threshold 的？

因为 RNA 配对有物理约束——一个碱基最多配一个。如果直接取所有高分位置会有大量冲突。Budget + Top-K 保证了：
1. 预测总数合理（不会过预测）
2. 优先选最确定的配对
3. 隐式避免了一碱基多配的问题（高分位置通常不冲突）
