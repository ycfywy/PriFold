# v4 Loss 计算详解（附具体数值示例）

> 本文档以一个具体的 bpRNA 样本为例，逐步拆解 v4 模型训练时 loss 是如何计算的。

---

## 0. 整体公式

```
total_loss = bce_flow          # 主损失：flow logit 的 adaptive focal BCE
           + 0.05 × stack      # 堆叠连续性
           + 0.02 × nc         # 非交叉约束
           + 0.2  × density    # 密度回归
           + 0.3  × bce_direct # direct head 的 adaptive focal BCE（无时间加权）
           + 0.05 × pair_count # direct head 配对数校准
```

---

## 1. 举个具体例子

### 输入样本

假设一条 bpRNA 序列：

```
序列长度 L = 100
Ground truth contact map x₁: 100×100 对称矩阵，共 25 个配对（50 个 1）
总有效 pixel 数 = 100×100/2 - 对角线约束 ≈ 4,753 个（|i-j| ≥ 3 的上三角）
正样本: 25 个 pixel 是 1
负样本: 4,728 个 pixel 是 0
正负比: 1:189
```

### Step 1: 采样时间 t 和噪声 x_t

```python
t = 0.7  # 随机采样，假设这个 batch 中这条序列抽到 0.7

# 生成 x_t（Bernoulli noise schedule）
# 对于 x₁=1 的位置: P(x_t=1) = t×1 + (1-t)×0.005 = 0.7015
# 对于 x₁=0 的位置: P(x_t=1) = t×0 + (1-t)×0.005 = 0.0015

# 期望结果:
#   25 个 GT=1 的位置中，约 17-18 个在 x_t 中仍为 1
#   4728 个 GT=0 的位置中，约 7 个在 x_t 中意外为 1
```

**含义**：模型看到的 `x_t` 是一个"70% 接近真值"的带噪 contact map。模型的任务是从这个噪声版本推断出完整的 `x₁`。

---

### Step 2: 模型前向

模型输入 `x_t` 和 `t=0.7`，加上 MARS 条件特征，产出三个东西：

```
flow_logit:    (100, 100) — 对 x₁ 每个位置的预测 logit
direct_logit:  (100, 100) — 不依赖 t 的直接评分 logit
density_pred:  标量 — 预测的每碱基配对密度
```

假设模型对某些代表性 pixel 的预测：

| 位置 | GT (x₁) | flow_logit | σ(flow_logit) | direct_logit | σ(direct_logit) |
|------|----------|------------|---------------|--------------|-----------------|
| (10,50) 真配对 | 1 | 2.5 | 0.924 | 2.0 | 0.881 |
| (20,60) 真配对 | 1 | 0.3 | 0.574 | 0.1 | 0.525 |
| (30,70) 真配对（难） | 1 | -0.5 | 0.378 | -0.8 | 0.310 |
| (15,40) 非配对 | 0 | -4.0 | 0.018 | -3.5 | 0.029 |
| (25,80) 非配对（FP） | 0 | 1.0 | 0.731 | 0.8 | 0.690 |

---

### Step 3: 计算 Adaptive pos_weight

```python
gt_pairs = 25
L_eff = 100
pair_per_base = 25 / 100 = 0.25
alpha = min(0.25 / 0.5, 1.0) = 0.5

pos_weight = pos_weight_min + alpha × (pos_weight_base - pos_weight_min)
           = 10 + 0.5 × (199 - 10)
           = 10 + 94.5
           = 104.5
```

**含义**：这条序列密度适中，positive 位置的 loss 被放大 104.5 倍（相比 negative 的 1 倍）。

---

### Step 4: 计算 Flow BCE Loss（主损失）

对每个 pixel (i,j)：

```
原始 BCE(i,j) = -[pos_weight × x₁ × log(σ(logit)) + (1-x₁) × log(1-σ(logit))]
```

#### 逐个计算示例 pixel：

**(10,50) 真配对，预测好 (logit=2.5, σ=0.924)**
```
raw_bce = -[104.5 × 1 × log(0.924) + 0] = -[104.5 × (-0.079)] = 8.26
focal_w = (1 - 0.924)^2 = 0.0058
bce_focal = 8.26 × 0.0058 = 0.048
time_w = 1/(1 - 0.7×0.995) = 1/0.304 = 3.29
bce_final = 0.048 × 3.29 = 0.158
```

**(20,60) 真配对，预测一般 (logit=0.3, σ=0.574)**
```
raw_bce = -[104.5 × 1 × log(0.574)] = -[104.5 × (-0.555)] = 58.0
focal_w = (1 - 0.574)^2 = 0.182
bce_focal = 58.0 × 0.182 = 10.55
time_w = 3.29
bce_final = 10.55 × 3.29 = 34.7
```

**(30,70) 真配对，预测错 (logit=-0.5, σ=0.378)**
```
raw_bce = -[104.5 × 1 × log(0.378)] = -[104.5 × (-0.972)] = 101.6
focal_w = (1 - 0.378)^2 = 0.387
bce_focal = 101.6 × 0.387 = 39.3
time_w = 3.29
bce_final = 39.3 × 3.29 = 129.3
```

**(15,40) 非配对，预测对 (logit=-4.0, σ=0.018)**
```
raw_bce = -[0 + 1 × log(1-0.018)] = -log(0.982) = 0.018
focal_w = (1 - 0.982)^2 = 0.0003
bce_focal = 0.018 × 0.0003 = 0.0000054
time_w = 3.29
bce_final ≈ 0.000018
```

**(25,80) 非配对但误判 (logit=1.0, σ=0.731)**
```
raw_bce = -[0 + 1 × log(1-0.731)] = -log(0.269) = 1.31
focal_w = (1 - 0.269)^2 = 0.534
bce_focal = 1.31 × 0.534 = 0.70
time_w = 3.29
bce_final = 0.70 × 3.29 = 2.30
```

#### 对比一览

| Pixel | 类别 | 预测质量 | BCE 贡献 |
|-------|------|---------|----------|
| (10,50) | 正样本 | 好 (σ=0.92) | **0.16** |
| (20,60) | 正样本 | 一般 (σ=0.57) | **34.7** |
| (30,70) | 正样本 | 差 (σ=0.38) | **129.3** |
| (15,40) | 负样本 | 好 (σ=0.02) | **0.00002** |
| (25,80) | 负样本 | 差 (σ=0.73) | **2.30** |

**关键观察**：
- 正确预测的负样本贡献 ≈ 0（被 focal 压到几乎为零）
- 正确预测的正样本贡献也很小（focal 压制 easy positive）
- **真正贡献 loss 的是那些"困难"的正样本和 false positive**

#### 最终 flow BCE（在所有 pixel 上平均）

```
假设 25 个正样本中：
  - 10 个预测好 (avg 贡献 ~0.2)       → 总 ~2
  - 10 个预测一般 (avg 贡献 ~30)      → 总 ~300
  - 5 个预测差 (avg 贡献 ~100)        → 总 ~500

4728 个负样本中：
  - 4700 个预测好 (avg 贡献 ~0.00005) → 总 ~0.2
  - 20 个预测一般 (avg 贡献 ~0.5)     → 总 ~10
  - 8 个 false positive (avg 贡献 ~2) → 总 ~16

总 loss sum ≈ 828
总 pixel 数 ≈ 4753
bce_flow = 828 / 4753 ≈ 0.174
```

---

### Step 5: Stacking Loss（0.05）

```python
# 检查堆叠连续性：如果 (i,j) 是配对，(i+1,j-1) 也应该是
prob = sigmoid(flow_logit)

# 对每个 (i,j)，计算 prob[i,j] × (1 - prob[i+1,j-1])
# 即："你预测了 (i,j) 但没预测相邻堆叠"的惩罚

stack_loss = 0.05 × avg_discontinuity
```

假设 stack_loss ≈ 0.05 × 0.3 = **0.015**

---

### Step 6: Non-Crossing Loss（0.02）

```python
# 每个碱基最多配对 1 个伙伴
prob = sigmoid(flow_logit)
row_sum = prob.sum(dim=-1)    # 每行（每个碱基）的总配对概率
excess = relu(row_sum - 1.0)  # 超过 1 的部分

nc_loss = 0.02 × excess.mean()
```

假设 nc_loss ≈ 0.02 × 0.2 = **0.004**

---

### Step 7: Density Loss（0.2）

```python
gt_density = gt_pairs / L_eff = 25 / 100 = 0.25
density_pred = 模型预测值，假设 = 0.30

density_loss = 0.2 × SmoothL1(0.30, 0.25) = 0.2 × 0.05 = 0.01
```

---

### Step 8: Direct BCE Loss（0.3）

和 flow BCE 完全一样的公式，但：
- 用 `direct_logit` 代替 `flow_logit`
- **没有时间加权 time_w**（因为 direct head 不依赖 t）

```
direct_bce ≈ 0.174 / 3.29 ≈ 0.053（去掉 time weight 后更小）
direct_loss = 0.3 × 0.053 ≈ 0.016
```

---

### Step 9: Pair Count Loss（0.05）

```python
# 从 direct_logit 计算预测的总配对数
pred_pairs = sigmoid(direct_logit).sum() / 2  # 假设 = 35
pred_density = 35 / 100 = 0.35
gt_density = 0.25

pair_count_loss = 0.05 × SmoothL1(0.35, 0.25) = 0.05 × 0.10 = 0.005
```

---

### Step 10: 最终汇总

| 分量 | 值 | 占比 |
|------|------|------|
| bce_flow (主) | 0.174 | **78.8%** |
| stack_loss | 0.015 | 6.8% |
| nc_loss | 0.004 | 1.8% |
| density_loss | 0.010 | 4.5% |
| direct_loss | 0.016 | 7.2% |
| pair_count_loss | 0.005 | 2.3% |
| **total_loss** | **0.221** | 100% |

## 2. 为什么训练后期 total_loss 会降到 0.01？

上面的例子是训练初期的情况。训练 200+ epoch 后：

```
大部分正样本的预测变好了:
  - 原来 10 个"预测一般" → 变成"预测好"
  - 原来 5 个"预测差" → 变成"预测一般"

false positive 减少了:
  - 原来 8 个 → 变成 3 个

但 focal loss 会把"预测好"的贡献压到接近 0:
  σ=0.9 → focal_w = (1-0.9)^2 = 0.01
  σ=0.95 → focal_w = (1-0.95)^2 = 0.0025
```

**结果**：

```
25 个正样本：
  - 20 个预测好 (σ>0.85): focal_w < 0.02 → 每个贡献 <0.5 → 总 ~10
  - 4 个预测一般 (σ~0.6-0.8): focal_w ~0.05-0.16 → 每个贡献 ~5-20 → 总 ~40
  - 1 个预测差 (σ<0.5): 贡献 ~80

4728 个负样本：
  - 几乎全对 (σ<0.05): focal_w ≈ 0 → 总 ~0.5
  - 3 个 FP (σ>0.5): 总 ~5

总 loss sum ≈ 136
bce_flow = 136 / 4753 ≈ 0.029

其他分量也相应下降：
  stack: ~0.005, nc: ~0.002, density: ~0.002, direct: ~0.005, pair_count: ~0.002
  
total ≈ 0.029 + 0.005 + 0.002 + 0.002 + 0.005 + 0.002 = 0.045
```

再后期 (epoch 248)，模型更好了：

```
total ≈ 0.009-0.010
```

## 3. 为什么 loss=0.01 但 F1 只有 0.49？

### 关键 insight

**loss=0.01 不代表模型全对了。它代表 focal loss 把"做对了的"压到了 0，只剩下少量难样本在贡献 loss。**

做个极端假设：

```
假设模型对 25 个正边中的 15 个预测 σ > 0.9（好）
剩下 10 个预测 σ ∈ [0.4, 0.7]（不好）

Focal loss 下：
  - 那 15 个好的：贡献 ≈ 0（被 focal 压掉了）
  - 那 10 个差的：贡献了几乎全部 loss

→ loss 看起来"低"（因为好的被 mask 掉了）
→ 但 F1 不高（因为 10/25 = 40% 的正边预测质量不够，threshold 后 miss 了）
```

### 另一个视角：projection 阈值的影响

即使模型预测 σ=0.6，这已经让 BCE loss 很低了。但在 projection 时：
- `score_threshold=0.5`：σ=0.6 会被选入候选
- `default_budget_fraction=0.35`：如果候选太多，只保留 top 35%
- 很多 σ≈0.55-0.65 的"模糊边"可能被选入候选，但其中很多是 false positive

**所以"loss 低"和"F1 高"之间的 gap，本质是 BCE 连续概率空间和 F1 离散结构空间之间的语义鸿沟。**

## 4. 各分量的实际贡献分析

### 训练后期各分量的典型值（从实际 log 估算）

```
total_loss ≈ 0.010

其中：
  bce_flow:       ~0.005-0.006  (50-60%)  ← 仍然是主导
  direct_loss:    ~0.002-0.003  (20-30%)  ← 0.3× 的 direct BCE
  stack_loss:     ~0.001        (10%)
  density_loss:   ~0.0005       (5%)
  nc_loss:        ~0.0003       (3%)
  pair_count_loss:~0.0002       (2%)     ← ⚠️ 权重 0.05 太弱！
```

### 问题诊断

1. **pair_count_loss 贡献太小**：只占 total 的 2%。pred/gt=1.47 的过预测问题，每 epoch 只能获得 ~0.0002 的梯度信号来纠正——这几乎不可能让模型学会控制预测数量。

2. **bce_flow 被 focal+time 过度压制**：Focal γ=2 + 时间加权，让大部分"还行但不完美"的预测贡献大幅下降。模型对这些 "border case" 的优化动力不足。

3. **stack 和 nc 起不到太大作用**：权重太小（0.05, 0.02），而且它们优化的是结构合理性（堆叠、非交叉），不直接优化 "正确预测了哪些配对"。

---

## 5. 总结：loss 信号不足的根因

```
                    ┌─────────────────────────────┐
                    │  Total Loss = 0.010          │
                    └──────────────┬──────────────┘
                                   │
          ┌────────────────────────┼────────────────────────┐
          │                        │                        │
   4728 个 easy negative      15 个 easy positive      10 个 hard positive
   贡献: ≈0 (focal killed)    贡献: ≈0 (focal killed)  贡献: ~0.008 (全部有效信号)
          │                        │                        │
          │                        │                        ▼
          │                        │               这 10 个边的预测质量
          │                        │               决定了 F1 能不能涨
          ▼                        ▼                        
   模型已经学会说"不"       模型已经学会说"是"      但还有很多边界 case
   → 无信息量               → 无信息量              → 仅存的梯度来源
```

**一句话**：focal loss 成功地让模型聚焦于难样本，但代价是总 loss magnitude 太小，优化器的 step size（lr × gradient）也跟着变小了。这就是为什么 "LR 8e-5 好像不够用"——不是 LR 小，是 gradient 被 focal 压得太小。
