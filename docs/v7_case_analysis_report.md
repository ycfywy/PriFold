| F1=0 rate | 0.5% | 3.8% | 7.6x |

**这个 gap 说明**：
- 模型对训练集中见过的 RNA 可以很好地预测（F1=0.81）
- 但对从未见过的 RNA 家族（val/test 中的稀有 RFAM 家族），预测能力大幅下降
- 这不是典型的过拟合（val loss 没有上升），而是**泛化能力不足**——训练数据的多样性不够覆盖所有 RNA 折叠模式

## 6. 四种失败模式分析

![Failure Mode Analysis](../symfold/outputs/v7_full/case_analysis/failure_mode_analysis.png)

对 427 个 bad case (F1<0.3) 的分析：

### Mode 1: 数量对但位置错 (36.5%, N=156) ⭐ 最大问题

- 特征：0.7 < pred/gt < 1.3, 但 F1 < 0.3
- 平均长度：161，密度：0.253
- avg pred/gt = 1.01（数量几乎完美匹配！）
- near_miss_pct = 0.109（大部分 FP 离 GT 很远）

**根因**：模型学会了"正确的配对数量"但没学会"正确的位置"。对于长序列中结构复杂的 RNA，positional reasoning 能力不足。

### Mode 2: 完全位置错误 (18.3%, N=78)

- 特征：F1=0, tp=0, 但有预测
- 平均长度：97，密度：0.166（偏低）
- avg pred/gt = 2.28（过预测）
- near_miss_pct = 0.046（几乎没有近似正确的预测）

**根因**：模型对这类 RNA 完全无知，预测出"看似合理但完全错误"的结构。

### Mode 3: 偏移预测 (15.7%, N=67) ⭐ 新发现

- 特征：near_miss_pct > 0.3（30%+ 的 FP 在 GT 附近 ±3 位置内）
- 平均长度：104，密度：0.187
- avg pred/gt = 1.20
- **avg near_miss_pct = 0.460**（46% 的预测配对在 GT 附近！）

**这就是用户提到的"平移"问题**：模型预测的 stem 结构位置偏移了几个 nucleotide。结构模式识别正确，但精确定位能力不足。

### Mode 4: 严重过预测 (7.7%, N=33)

- 特征：pred/gt > 2
- 平均长度：132，密度：**0.076**（极低密度）
- avg pred/gt = 3.54

**根因**：低密度 RNA 的 density head calibration 不准。

### Mode 5: 其他 (20.6%, N=88)

混合模式。

## 9. 核心问题：为什么仍有 F1=0

详见 [v7_f1_zero_loss_analysis.md](./v7_f1_zero_loss_analysis.md)。

**结论**：

1. **Loss 不对称**：pos_weight 只惩罚 FN，不惩罚 FP；FP 被海量 TN 稀释
2. **平移预测的 loss 景观平坦**：偏移 1 位和偏移 50 位的 loss 几乎一样
3. **Pair count / ratio penalty 只看数量**：对"数量对但位置错"的 case 无信号
4. **训练数据多样性不足**：稀有 RFAM 家族样本太少

**推荐改进**：
- OHEM (Online Hard Example Mining)：只取 top-k hardest negatives 计算 neg loss
- FP 加权：给 false positive 位置额外惩罚
- 碱基配对兼容性约束：利用 AU/GC/GU 规则减少不可能配对
