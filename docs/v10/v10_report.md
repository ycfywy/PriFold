# v10 版本报告：MARS 解冻实验

## 1. 版本定位

v10 的**唯一改动**是将 MARS 预训练语言模型（160M 参数）从冻结改为**全部解冻**参与训练。模型架构（DensityNetProPlus）与 v9 完全相同。

| | v9 | v10 |
|---|---|---|
| MARS 权重 | **冻结** | **解冻** |
| 可训练参数 | 5.09M | **165.7M** |
| MARS LR | — | 5e-6 |
| Head LR | 5e-4 | 5e-4 |
| 初始化 | 从头训练 | 从 v9 best.pt warm-start |
| Grad Clip | 1.0 | 0.5（保护 MARS） |
| 训练轮次 | 183 epochs | 151 epochs |

### 2.2 Train / Val / Test 对比（v10 最终）

| 指标 | Train | Val | Test |
|------|-------|-----|------|

- 模型在训练集上近乎完美，但泛化到新数据时下降 18pp

## 3. 放开 MARS 权重的效果分析

### 3.1 直接增益

| 维度 | v9 → v10 | 说明 |
|------|----------|------|
| Test F1 | 0.6961 → **0.7287** (+3.26pp) | 显著提升 |
| Val F1 | 0.6814 → **0.7265** (+4.51pp) | 更大提升 |
| 训练参数 | 5M → **166M** (33x) | 代价是巨大参数量 |

### 3.2 按家族分析 — MARS 解冻对谁帮助最大？

| 家族 | Train F1 | Test F1 | 泛化 Gap | 说明 |
|------|----------|---------|---------|------|
| CRW | 0.9945 | 0.9398 | 0.055 | 几乎饱和，gap 极小 |
| SPR | 0.9668 | 0.9530 | 0.014 | 最佳家族 |
| RNP | 0.9135 | 0.7951 | 0.118 | 中等 gap |
| tmRNA | 0.9270 | 0.7420 | 0.185 | 较大 gap |
| **RFAM** | 0.9034 | **0.7056** | **0.198** | 主要瓶颈 |
| **SRP** | 0.9228 | **0.5789** | **0.344** | 泛化最差 |

**关键发现**：
- RFAM（占 87% 样本）的泛化 gap 最大（0.198），是整体性能的瓶颈
- CRW/SPR 家族已经接近饱和，MARS 解冻主要帮助了 RFAM 的提升
- SRP 泛化 gap 达到 0.344，可能需要更多数据或特殊处理

### 3.3 MARS 解冻的机制分析

v9（MARS frozen）时，MARS 输出的 hidden states 和 attention maps 是固定的"通用 RNA 表征"。v10 解冻后：

1. **特征适应**：MARS 的 attention 学会了更关注碱基配对相关的模式，而非通用语言模型特征
2. **表征精调**：hidden states 从"哪些碱基相似"优化为"哪些碱基可能配对"
3. **联合优化**：MARS 和下游 head 联合训练，梯度流贯通 → 端到端优化

**代价**：
- 训练时间增加（参数量 33x），需要更大显存
- 分层 LR 必须精心设置（mars_lr=5e-6 远小于 head_lr=5e-4），否则破坏预训练知识
- 需要 grad_clip=0.5 保护 MARS（v9 用 1.0）

| 长度区间 | 样本数 | Test F1 | Precision | Recall | Bad Rate |
|----------|--------|---------|-----------|--------|----------|
| 0-50 | 29 | 0.7046 | 0.6571 | 0.7802 | 17.2% |
| **50-100** | 542 | **0.7662** | 0.7505 | 0.7986 | 7.2% |
| 100-150 | 393 | 0.7135 | 0.7134 | 0.7245 | 8.9% |
| 150-200 | 139 | 0.6687 | 0.6967 | 0.6536 | 10.1% |
| 200-300 | 97 | 0.7075 | 0.7537 | 0.6725 | 7.2% |

| 400-500 | 27 | 0.6251 | 0.7227 | 0.5555 | 14.8% |

**规律**：
- 短序列 (50-100) 表现最好：结构简单，配对模式有限
- 超长序列 (400-500) 表现最差：远距离依赖多，且样本稀少
- 长序列的 Recall 明显下降（L>200: R≈0.67 vs L<100: R≈0.80）

## 5. Bad Cases 分析

### 108 个 Bad Cases (F1 < 0.3) 的特征

| 特征 | 值 |
|------|-----|
| 总数 | 108 (占 test 8.3%) |
| 家族分布 | RFAM: 105, SRP: 2, CRW: 1 |
| F1=0 的样本 | 34 个 (完全预测错位置) |
| 平均长度 | 136 (与全体相同) |
| 过度预测 (pred/GT>1.5) | 19 (17.6%) |
| 数量对但位置全错 | 83 (**76.9%**) |
| Shifted FP 占总 FP | **47.1%** |

### 核心发现

**99/108 (91.7%)** 的 bad case 在训练集中有高度相似的样本——模型见过类似结构但仍然学不会。这说明瓶颈是**模型能力**而非数据覆盖。

## 6. 关键结论

### 放开 MARS 权重的影响

| 方面 | 结论 |
|------|------|
| ✅ 性能提升 | Test F1 +3.26pp (0.6961→0.7287)，显著有效 |
| ✅ 全面改善 | Precision/Recall/MCC 均有提升 |
| ✅ 主要受益者 | RFAM 家族（占 87%），泛化 gap 从 v9 的更大值缩小 |
| ⚠️ 过拟合加剧 | Train F1=0.91 vs Test F1=0.73, gap=0.18 |
| ⚠️ 残余问题 | 47% FP 是位置偏移、108 个 bad case 未解决 |
| ❌ 长序列瓶颈 | L>400 的样本 F1 仍然很低(0.625) |

### v10 的贡献与局限

**贡献**：
- 证明了 MARS 微调对 RNA 结构预测有显著帮助
- 建立了分层 LR + warm-start 的训练范式
- 将 state-of-the-art 从 0.70 推到 0.73

**局限**：
- 过拟合严重（Train 0.91 vs Test 0.73）
- Bad cases 的根因是模型给 GT 配对的概率极低（mean=0.21），特征表达力不足
- 47% 的 FP 是位置偏移 ±1，定位精度仍有改善空间
- 需要更多正则化或更好的架构来缩小泛化 gap
