# v11 改进方案提案

> 基于 v10 全面分析结果，提出候选改进方向。每次实验只引入一个变量，逐步验证。

## 0. v10 基线总结

| 指标 | 值 |
|------|-----|
| Test F1 | 0.7287 |
| Train F1 | 0.9110 |
| 泛化 Gap | 0.1823 |
| Bad cases (F1<0.3) | 108/1303 (8.3%) |
| Bad case 主因 | 42.6% 模型完全不认识结构，56.5% 有感知但不够自信 |
| FP 平均概率 | 0.64（高度自信的错误） |
| Shifted FP 占比 | 47.1% |

---

## 1. 改进大分类

### 类别 A：数据增强（增加训练数据多样性）

改变输入序列或标签，让模型见到更多变化，减少过拟合。

| 编号 | 方法 | 描述 | 改什么 | 预期效果 |
|------|------|------|--------|----------|
| A1 | 协变替换加强 | 将 select 0.2→0.5, replace 0.4→0.6 | 序列 | 更多序列变体，强迫模型学结构而非记序列 |
| A2 | 非配对位随机突变 | 对非配对位置随机替换碱基 5-15% | 序列 | 破坏序列记忆，增强序列鲁棒性 |
| A3 | 子结构 dropout | 随机删除 GT 中 10-20% 配对 | 标签 | 防止对特定拓扑过拟合 |
| A4 | 随机 crop | 截取子序列（如 60-80% 长度）+ 对应子矩阵 | 序列+标签 | 增加短序列训练量，变相数据扩增 |
| A5 | 接触图噪声 | 在 GT 中随机添加 1-3% 假配对（soft label 0.3） | 标签 | 降低模型过度自信 |

### 类别 B：正则化（抑制过拟合）

不改变数据，改变训练策略来限制模型容量。

| 编号 | 方法 | 描述 | 预期效果 |
|------|------|------|----------|
| B1 | 增大 dropout | 0.2 → 0.3 或 0.35 | 直接降低过拟合 |
| B2 | 增大 drop_path | 0.15 → 0.25 | 随机跳过 transformer 层，减少共适应 |
| B3 | Label smoothing | GT=1 位置周围 ±1 赋值 0.2-0.3 | 缓解位置偏移问题 + 降低 FP 自信度 |
| B4 | Weight decay 增大 | 0.02 → 0.05 | 限制参数范数 |
| B5 | R-Drop / 对称 KL | 同一输入前向两次，对齐两次输出 | 鼓励一致性 |

### 类别 C：采样策略（改变训练数据的组成）

不改变单个样本，改变哪些样本被更多地训练。

| 编号 | 方法 | 描述 | 预期效果 |
|------|------|------|----------|
| C1 | Hard-case 过采样 | 对训练集中与 bad case 结构相似的样本 2-3x 过采样 | 让模型多学困难模式 |
| C2 | 家族平衡采样 | 降低 RFAM-easy 权重，提升小家族比例 | 减少对 easy pattern 的偏向 |
| C3 | 扩充数据集 | 加入 RNAStrAlign 联合训练 | 绝对数据量增加 |
| C4 | 课程学习 | 先训短+高密度样本，逐步加长+低密度 | 从简入难，稳定训练 |

### 类别 D：模型/解码改进

改变模型结构或推理方式。

| 编号 | 方法 | 描述 | 预期效果 |
|------|------|------|----------|
| D1 | 多层融合 | MARS hidden layers 加浅层 [1,2,3,6,9,12] | 浅层保留更多局部结构信息 |
| D2 | 辅助任务 | 加 per-residue 配对状态预测 loss | 强迫 MARS 编码结构信息 |
| D3 | Score map 平滑 | 推理时 Gaussian smooth → 取极大值 | 缓解 ±1 偏移 |
| D4 | Matching 解码 | 替换 topk 为贪心一对一匹配 | 保证每碱基最多一配对 |
| D5 | Confidence penalty | 对高概率 FP 加额外惩罚 loss | 抑制自信的错误 |

### 类别 E：Loss 改进

改变损失函数的设计。

| 编号 | 方法 | 描述 | 预期效果 |
|------|------|------|----------|
| E1 | 加大 OHEM 比例 | neg_ratio 3 → 5-6 | 更多 hard negative 训练 |
| E2 | 结构 consistency loss | 预测矩阵应满足 one-to-one 约束 | 减少非法预测 |
| E3 | 对比学习 | 同家族 RNA 的 score map 应相似 | 学习家族共性 |
| E4 | Focal gamma 调大 | 1.0 → 2.0 | 更关注难分类位置 |

---

## 2. 实验规划原则

1. **每次实验只引入一个变量**，其余保持与 v10 baseline 一致
2. **每个实验跑完后评估 val F1 + test F1**，确认方向
3. **有效的改进可以叠加**，但叠加时也要控制：A+B 验证后再加 C
4. **优先测试成本低、预期收益高的改进**

---

## 3. 建议实验顺序

根据「实现成本 × 预期收益 × 与根因的关联度」排序：

| 优先级 | 实验 | 理由 |
|--------|------|------|
| 1 | **A3 子结构 dropout** | 直接对抗"过拟合特定结构拓扑"，实现简单（改 GT 即可） |
| 2 | **B1 增大 dropout** | 最简单的正则化，改一个参数即可验证 |
| 3 | **A2 非配对位随机突变** | 破坏序列记忆，实现简单 |
| 4 | **B3 Label smoothing** | 已有实现基础，对 ±1 偏移问题针对性强 |
| 5 | **C1 Hard-case 过采样** | 针对 bad case 的直接干预 |
| 6 | **D1 多层融合** | 可能提升 MARS 表征对结构的捕获 |
| 7 | **E1 OHEM 加大** | 减少"自信的错误"，改一个参数 |

---

## 4. 实验基准设定

所有 v11 实验统一条件：

- 起点：v9 best.pt（不是 v10，保证公平对比）
- freeze_mars: false
- MARS LR: 5e-6, Head LR: 5e-4
- Epochs: 100
- 评估方式：val F1 (每 epoch) + test F1 (每 20 epoch)
- 对比 baseline: v10 在相同 epoch 数下的表现

---

## 5. 命名规范

```
v11a — A3 子结构 dropout
v11b — B1 增大 dropout
v11c — A2 非配对位随机突变
v11d — B3 Label smoothing
v11e — C1 Hard-case 过采样
v11f — D1 多层融合
v11g — E1 OHEM 加大
```

每个子实验独立跑，最后选取有效的组合为 v11-final。
