## 3. 数据特点分析

### 3.1 极端稀疏性

bpRNA 最显著的特点是 **contact map 极其稀疏**：

```
平均序列长度: 134 nt
平均配对数: 30.7 对
Contact map 大小: 134 × 134 = 17,956 个 pixel
正样本 (配对): 30.7 × 2 = 61 个 pixel（对称矩阵，每对出现两次）
有效上三角像素（|i-j|≥3）: ~8,800 个
正负比: 30.7 / 8800 ≈ 1:286

→ 99.6% 的输出位置是 0
→ 只有 0.4% 是配对
```

### 3.2 配对率分布

- 每碱基配对率 (pairs/L) 的分布集中在 **0.15-0.35** 之间
- 峰值在 ~0.24（即大约一半的碱基参与配对）
- 极低配对率 (<0.10): 9.2% 的序列
- 这些低配对率序列是模型最容易过预测的对象

### 3.3 长度 vs 配对的线性关系

配对数和序列长度高度线性相关：`n_pairs ≈ 0.23 × L`

```
斜率 ≈ 0.23 → 每个碱基平均参与 0.46 次配对（因为每对涉及 2 个碱基）
→ 约 54% 的碱基不参与任何配对
```

这意味着 contact map 的 density（pairs/L²）会随 L 增加而**急剧下降**：
- L=50: density ≈ 0.023/50 = 0.009
- L=100: density ≈ 0.023/100 = 0.005
- L=300: density ≈ 0.023/300 = 0.0015
- L=500: density ≈ 0.023/500 = 0.0009

**长序列的 contact map 更稀疏，对模型更难。**

### 3.4 未配对碱基比例

平均 54.6% 的碱基不参与配对（unpaired fraction 分布在 0.3-0.9 之间）。

部分序列未配对率极高（>70%）：占 16.1%。这些序列只有少量 stem-loop 结构，大部分碱基是自由的。

### 3.5 配对距离

- 平均配对距离：44.8 nt
- 大部分配对是短程的（<50 nt），形成 stem-loop 结构
- 但也有长程配对（100+ nt），形成复杂的多分支结构

### 3.6 GC 含量

- 均值 0.48，标准差 0.10
- 相对均匀，没有极端偏差
- A: 25.8%, G: 25.8%, U: 25.2%, C: 22.9%
- 近似均匀分布（略偏少 C）

## 4. 一个碱基能否配对多个碱基？

### 答案：**不能。bpRNA 严格执行 1:1 配对约束。**

验证结果：
```
遍历全部 24,233 条序列：
→ 一个碱基有 >1 个配对伙伴的序列数: 0
→ 每个碱基最多配对 1 个伙伴（max_partners=1）
→ 12 条序列完全无配对（全是 "."）
```

### 解释

RNA 二级结构的物理约束：
- 每个碱基通过 **Watson-Crick 配对**（A-U, G-C）或 **wobble 配对**（G-U）与另一个碱基形成氢键
- 一个碱基只能同时和一个碱基形成稳定的碱基对
- 这是热力学决定的，不是数据标注的人为限制

### 伪结（Pseudoknot）不是多重配对

bpRNA 中 9.2% 的序列含伪结（用 `[]` 或 `{}` 标注）。伪结是指**交叉配对**：

```
正常:      (((...)))         # i-j 嵌套
伪结:      (([...)]...)      # (i,j) 和 [k,l] 交叉：i < k < j < l
```

伪结中每个碱基**仍然只有一个配对伙伴**。不同的括号类型只是标记"哪些配对和哪些配对交叉"，不是标记多重配对。

### 对模型的含义

由于严格 1:1 配对：
- 理论上 contact map 每行/每列最多只有 1 个 "1"
- 模型的 `NonCrossingLoss`（nc_weight=0.02）就是约束这个：惩罚每行概率和 >1
- 这也解释了为什么 `default_budget_fraction=0.35` 可能偏高：对于 L=100 的序列，0.35×100=35 条边的预算，但实际只有 ~25 条边

## 5. 困难样本分析

### 5.1 哪些序列难？

| 类别 | 占比 | 为什么难 |
|------|------|---------|
| 低配对率 (pairs/L < 0.10) | 9.2% | 正负比极端（1:500+），过预测风险高 |
| 高未配对率 (>70%) | 16.1% | 大部分碱基不配对，模型必须学会"不预测" |
| 长序列 (>300nt) | 7.3% | map 更大更稀疏，长程配对更难学 |
| 含伪结 | 9.2% | 交叉配对打破嵌套结构假设 |

### 5.2 训练中的问题

这些困难类别在 train/val/test 中分布一致（8-10%），所以**不是分布偏移问题**。

真正的问题是：
1. 困难样本只占 10-16%，在 loss 中被大量简单短序列"淹没"
2. 短序列（50-100nt）占 42%，结构简单，模型很快学会 → loss 被它们拉低
3. 长序列和低密度序列的 loss 贡献被边缘化

## 6. 对 v4 训练的启示

### 6.1 为什么 train loss 这么低（0.01）？

- 85% 的序列长度 <200，配对结构相对简单
- 模型早期就学会了这些简单模式（贡献了 loss 下降的大头）
- focal loss 进一步压制了"已经做对"的部分
- 结果：aggregate loss 很低，但模型在困难样本上仍然表现不好

### 6.2 为什么 pred/gt ratio 始终 ~1.47？

- `pairs_per_base ≈ 0.23` 但模型预测 `≈ 0.34`（1.47×0.23）
- 模型学到了 "RNA 大约一半碱基会配对"（近似正确），但系统性偏高
- `pair_count_weight=0.05` 太弱，无法有效校准

### 6.3 为什么 val/test F1 不是 distribution shift 问题？

- 三个分割的分布几乎完全一致（KS test 差异极小）
- val F1 ≈ test F1，泛化 gap 不大
- 问题是模型的**绝对预测能力**不够，不是 train 和 test 不一样

### 6.4 实际瓶颈

```
真正的困难在于：
1. 极端稀疏性（1:286）→ BCE loss 信号被 easy negative 稀释
2. 过预测（pred/gt=1.47）→ 但 pair_count loss 太弱无法纠正
3. 长序列（7%）和低密度样本（9%）→ 贡献 loss 少但拖 F1
```

## 9. 结论

| 发现 | 对训练的影响 |
|------|-------------|
| Train/Val/Test 分布一致 | F1 低不是 distribution shift，是任务本身难 |
| 每碱基严格 1:1 配对 | NonCrossingLoss 约束合理，budget 可以更紧 |
| Contact map 极稀疏（0.4%） | Loss 被 easy negative 主导，有效梯度稀缺 |
| 85% 序列 <200nt | 简单样本 dominate loss，难样本学不够 |
| pred/gt=1.47 固化 | pair_count 约束太弱，需要加强 |
